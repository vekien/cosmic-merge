Cosmic Merge

A space-themed number merging game built entirely in one conversation with Claude.

🎮 How to Play

Tap any column to drop the current tile into it. Tiles with the same number that touch — horizontally or vertically — automatically merge and double. Chain reactions score big!

The game ends when the board is completely full with no valid moves left. A column that's full can still be played if your current tile matches the top tile there — they'll merge.

⭐ The Rainbow Tile

The ★ Rainbow tile appears with a 1% chance. When it lands next to a number, it multiplies that tile — 70% chance of ×2, 30% chance of ×4. It picks a random neighbour, so placement matters. The popup always shows you the before and after so there's no mystery.

🎲 Tile Randomness

Tiles use fixed weighted probabilities — nothing to do with what's on your board. We tried board-aware spawning (only tiles that exist on the board) but it made late-game frustrating when you'd have only 64s and 256s dropping constantly. Fixed weights feel fairer:

🧠 Smart Merging

The merge engine doesn't just find the first pair it can — it simulates every possible merge order and picks whichever one leads to the longest chain reaction. So if placing a tile could merge 2+2 into 4 then 4+4 into 8, it'll always find that path rather than stopping after the first merge.

This lookahead runs up to 6 levels deep so it stays snappy even on a full board.

↩ Undo

You get one undo charge at a time. It only works if your last placement caused no merges — once tiles start combining, the state is too changed to cleanly reverse. You earn a fresh undo charge every 1,000 points, and it never stacks above 1. The ring around the undo button shows your progress toward recharging.

💾 Auto-Save

Your game saves automatically after every move using your browser's localStorage. Close the tab, come back later — it picks up exactly where you left off. Hitting restart clears the save.

🛠 Behind the Scenes

This game was built entirely through a conversation with Claude — no code was written by hand. Here's how the design evolved:

  • Started simple — basic grid, tap to place, instant merges. No animations, flat colours.
  • Drop animation — the tile needed to feel physical. It now glides to the top of the chosen column then falls with a slight bounce, speed scaling with distance.
  • Next tile preview — originally shown to the right with a label. Moved to sit just left of the current tile, smaller and dimmed, so the visual hierarchy is obvious.
  • Glass/3D tiles — a layered CSS technique: inset highlight on the top edge, a gloss streak over the top half using a pseudo-element, and a dark gradient at the base to fake thickness.
  • Space theme — canvas starfield with twinkling and shooting stars, nebula radial gradients that slowly pulse, frosted glass UI panels, and column lane lines in the grid.
  • Smart merge algorithm — the first version was greedy (first pair wins). It missed chains like 2+2→4 then 4+4→8. The current version simulates all candidate merge orderings and scores each by chain depth before committing.
  • Stepped chain animations — merges originally resolved instantly. Now each wave animates separately: tiles slide to their merge destination, pop into the new value, then the next wave triggers after a short pause so you can follow what happened.
  • Combo multiplier — chain reactions multiply your score. Wave 1 scores at ×1, wave 2 at ×2, etc., with a banner popping up to celebrate.
  • Game over logic — originally ended when any tile reached row 0. Changed to: game over only when every cell is full AND no column can accept the current tile (including merging into a matching top tile).
  • Bomb & Hammer tiles — were added then removed. The bomb destroyed a random neighbour; the hammer smashed every tile in a column. Ultimately felt disruptive to the merge flow, so they were cut.
